Tintolab Decision Workspace v7

실행 방법
1. 폴더 안의 index.html을 브라우저에서 엽니다.
2. 새 항목 만들기 / 상태 변경 / 코멘트 추가 / JSON 내보내기를 사용합니다.
3. 데이터는 브라우저 localStorage에 저장됩니다.

이번 버전 주요 변경
- 상태 체계를 논의 / 방향합의 / 확정 3단계로 통일
- Item 상세 제목을 자동 높이 확장 textarea로 변경
- 좌측 LNB에 Item Tree 추가
- 도메인 = 트리 루트 구조로 통합
- 트리에서 아이템 선택 / 삭제 / 드래그 이동 지원
- 트리에서 도메인(트리) 생성 시 상세의 도메인 선택 목록과 동일 데이터 반영
- 상단 검색어는 좌측 트리와 Items 목록에 동시에 반영

Item Tree 사용법
1. 좌측 LNB의 검색창에 제목/코드/설명/담당자 등을 검색하면 해당 이슈만 표시됩니다.
2. 도메인(트리명)을 추가하면 즉시 Item 상세와 생성 모달의 도메인 목록에도 나타납니다.
3. 트리의 아이템을 다른 도메인 헤더로 드래그하면 도메인이 이동됩니다.
4. 트리의 × 버튼을 누르면 해당 아이템과 관련 코멘트/이력이 함께 삭제됩니다.

상태 규칙
- 신규 항목 기본 상태는 논의
- 확정 처리는 방향합의 상태에서만 가능
- 확정 해제 시 방향합의로 복귀
- 예전 상태값(Under Review, Agreed, Locked 등)은 자동으로 논의 / 방향합의 / 확정으로 정규화

엑셀 일괄등록 기능
1. 상단의 "엑셀 일괄등록" 버튼 클릭
2. 아래 두 방식 중 하나 사용
   - 엑셀 행 복사 후 붙여넣기
   - CSV UTF-8 저장 후 업로드
3. "미리보기"로 결과 확인
4. "일괄등록 실행" 클릭

업로드 규칙
- code가 기존 항목과 같으면 업데이트
- code가 없으면 신규 생성
- 확정 항목은 보호를 위해 건너뜀
- 제목(title)은 필수
- 권장 헤더:
  code, type, domain, title, description, priority, status, owner, dueDate, clientResponse, finalConfirmedValue
- 한글 헤더도 지원:
  코드, 유형, 도메인, 제목, 설명, 우선순위, 상태, 담당자, 마감일, 고객회신값, 최종확인값
- 상태값은 다음 3개만 사용:
  논의 / 방향합의 / 확정


추가된 탐색 화면
- LNB에 Item Tree 메뉴 추가
- Item Tree 화면에서 도메인(트리) 단위 탐색
- 검색, 도메인 생성, 드래그 이동, 삭제 지원
- 트리에서 선택한 항목은 우측 상세에서 바로 수정 가능


Item Tree Explorer v7
- LNB의 Item Tree 메뉴는 구조형 탐색 전용 화면입니다.
- 오른쪽 상세 편집 패널은 제거되었습니다.
- 도메인과 아이템이 라인 기반 위계 구조로 접기/펼치기 됩니다.
- 아이템에서 "상세보기"를 누르면 최종확인값이 트리 안에서 바로 열립니다.
- 열린 미리보기에서 "더보기"를 누르면 Items 탭의 해당 아이템으로 이동합니다.
- 아이템은 드래그로 다른 도메인으로 이동할 수 있습니다.
- × 버튼으로 아이템 삭제가 가능합니다.
- 트리 검색은 Item Tree 화면 안에서만 동작합니다.

Item Tree 도메인 관리
- 도메인(트리) 추가 오른쪽에서 관리할 도메인을 선택할 수 있습니다.
- ↑ / ↓ 버튼으로 도메인 순서를 변경할 수 있습니다.
- 삭제 시 해당 도메인 아이템은 다른 도메인으로 자동 이동되며 Items의 도메인 목록에도 동일하게 반영됩니다.


Item Tree v9
- 도메인 이름 변경 / 삭제 / 하위 도메인 생성 지원
- 도메인 드래그앤드롭으로 상하 이동 및 종속관계 변경
- 전체 접기 / 전체 펼치기 지원
- Items 탭 도메인 선택 목록에 계층 관계 표시


드래그앤드롭 안정화 수정
- Item Tree에서 도메인/아이템 드래그 후 화면 active 상태가 깨지지 않도록 수정
- drop/dragover 이벤트 전파를 정리해 예외 상황 발생 가능성을 낮춤
- 잘못된 드롭 대상 또는 자기 자신 대상 드롭은 무시하도록 보호

Item 목록 표시 정책
- 코드: 좌측 상단 작은 박스 표기
- 제목: 기존 정책 유지
- 설명: 목록 비노출
- 유형/도메인/우선순위/상태: 기존 정책 유지
